Que son los puertos de red TCP y UDP

Ishiro Isaac Angulo Roman 1C

Definicion de Puerto de Red

Un puerto es un punto virtual en el que comienzan y terminan las conexiones de red. Los puertos estan basados en software y los gestiona el sistema operativo de un ordenador. Cada puerto esta asociado a un proceso o servicio especifico.

Los puertos permiten a los ordenadores diferenciar facilmente los distintos tipos de trafico: los correos electronicos van a un puerto distinto de las paginas web, por ejemplo, aunque ambos lleguen a un ordenador a traves de la misma conexion a Internet. Mientras que las direcciones IP permiten que los mensajes vayan hacia y desde dispositivos especificos, los numeros de puerto permiten dirigirse a servicios o aplicaciones especificos en esos dispositivos.

Tipos de Puertos

Hay 65 535 numeros de puerto posibles, aunque no todos son de uso comun.

  • Puertos conocidos (Well-known ports): Van del 0 al 1023. Estan registrados y asignados por la Autoridad de Numeros Asignados de Internet (IANA) a servicios especificos y bien conocidos.
  • Puertos registrados (Registered ports): Van del 1024 al 49151. Estos son utilizados por aplicaciones y servicios que no son tan universales como los de los puertos conocidos.
  • Puertos dinamicos o privados: Van del 49152 al 65535. Estos puertos se utilizan para conexiones temporales y no estan asignados a ningun servicio en particular.

Protocolos TCP vs UDP

TCP (Protocolo de Control de Transmision)

Es uno de los protocolos fundamentales de Internet. Esta disenado para proporcionar una entrega de datos fiable, ordenada y verificada de errores.

  • Es "orientado a la conexion", lo que significa que establece una conexion formal (mediante un proceso de "saludo de tres pasos") antes de enviar los datos.
  • Garantiza que todos los paquetes lleguen en el orden correcto y sin errores. Si un paquete se pierde, TCP se encarga de retransmitirlo.
  • Es mas lento que UDP debido a esta sobrecarga de verificacion.
  • Se usa cuando la integridad de los datos es crucial (ej. navegacion web, correos electronicos, transferencia de archivos).

UDP (Protocolo de Datagramas de Usuario)

Es el otro protocolo principal de transporte. A diferencia de TCP, UDP es sin conexion y no garantiza la entrega, el orden o la integridad de los paquetes de datos.

  • Es "no orientado a la conexion"; simplemente envia los paquetes al destino sin un acuerdo previo.
  • Es mucho mas rapido que TCP porque no tiene la sobrecarga de verificar la conexion ni el orden de los paquetes.
  • Se usa cuando la velocidad es mas importante que la fiabilidad absoluta (ej. streaming de video, juegos en linea, llamadas de voz por IP), donde perder un pequeno paquete de datos es preferible a detener la transmision para esperar la retransmision.

Puertos utilizados por protocolos de red

HTTP (Protocolo de Transferencia de Hipertexto)

  • Puerto 80/TCP. Es el puerto utilizado por HTTP, el protocolo de transferencia de hipertexto que se utiliza para acceder a todas las paginas web.

HTTPS (Protocolo de Transferencia de Hipertexto Seguro)

  • Puerto 443/TCP. Es el puerto utilizado por HTTPS, el protocolo de transferencia de hipertexto seguro.

SMTP (Protocolo Simple de Transferencia de Correo)

  • Puerto 25/TCP. Historicamente, es el puerto estandar para la transferencia de correo (SMTP) entre servidores.
  • Puerto 587/TCP. Es el puerto predeterminado para el envio de correos desde un cliente (como Outlook o Gmail) a un servidor de correo. Utiliza cifrado TLS para asegurar la conexion.
  • Puerto 465/TCP (SMTPS). Fue un puerto utilizado para SMTP Seguro (SMTPS), aunque ahora se prefiere el 587.

POP3 (Protocolo de Oficina de Correos v3)

  • Puerto 110/TCP. Es el puerto POP3 predeterminado y no esta cifrado.
  • Puerto 995/TCP (POP3S). Es el puerto cifrado para POP3, que funciona a traves de TLS/SSL.

IMAP (Protocolo de Acceso a Mensajes de Internet)

  • Puerto 143/TCP. Este es el puerto IMAP predeterminado que no proporciona ningun cifrado.
  • Puerto 993/TCP (IMAPS). Es el puerto seguro para IMAP y funciona a traves de cifrado TLS/SSL.

MIME (Extensiones Multipropósito de Correo de Internet)

  • Nota: MIME no es un protocolo que utilice un puerto; es un estandar de formato.
  • MIME (Multipurpose Internet Mail Extensions) es un estandar que permite identificar diferentes tipos de informacion.
  • Es una mejora al estandar de correo electronico que permite anexar archivos que no son de texto (como imagenes, audio, video o documentos) a los correos. Lo hace definiendo encabezados como Content-Type y Content-Transfer-Encoding.

FTP (Protocolo de Transferencia de Archivos)

  • Puerto 21/TCP (Control). FTP utiliza dos conexiones. El puerto 21 se usa para el "canal de control", es decir, para enviar comandos (como autenticarse, listar archivos, etc.).
  • Puerto 20/TCP (Datos). Este puerto se utiliza para el "canal de datos", por donde se transfieren los archivos en si.

FTPS (FTP Seguro)

  • Puertos 21/TCP y 990/TCP. FTPS (FTP Secure) anade una capa de seguridad SSL/TLS a FTP. Puede operar en el puerto 21 (para conexiones explicitas) o en el puerto 990 (para conexiones implicitas).

SFTP (Protocolo de Transferencia de Archivos SSH)

  • Puerto 22/TCP. SFTP (SSH File Transfer Protocol) no es FTP sobre SSL; es un protocolo completamente diferente que transfiere archivos sobre el protocolo SSH (Secure Shell). Utiliza el mismo puerto que SSH, el 22, para todas las comunicaciones (tanto control como datos).

BitTorrent

  • Puertos 6881-6889/TCP (Historico). BitTorrent usaba tradicionalmente este rango de puertos para la comunicacion entre peers.
  • Puertos dinamicos/alternativos. Los clientes modernos de BitTorrent suelen usar puertos en rangos mas altos (como 50000 o mas) y a menudo seleccionan un puerto al azar o permiten al usuario configurarlo, para evitar bloqueos.

FastTrack

  • Puerto 1214/TCP (Historico). Este era el puerto asociado con los clientes de la red FastTrack, como Kazaa, Morpheus y Grokster.
  • Nota: Versiones posteriores de este protocolo P2P fueron disenadas para no usar un puerto fijo y podian incluso usar puertos comunes (como el 80) para saltar firewalls.

IRC (Internet Relay Chat)

  • Puerto 6667/TCP (Estandar de facto). Aunque el puerto original asignado era el 194, el estandar de facto (el mas usado en la practica) siempre ha sido el 6667.
  • Puertos 6660–6669/TCP. Es comun que las redes de IRC utilicen este rango de puertos cercanos.

XMPP (Protocolo Extensible de Mensajeria y Presencia)

  • Puerto 5222/TCP. Es el puerto estandar para la comunicacion de cliente a servidor.
  • Puerto 5269/TCP. Es el puerto utilizado para la comunicacion de servidor a servidor.
  • Puerto 5223/TCP. Se utiliza a veces para conexiones antiguas con SSL.

MTProto (Protocolo Movil de Telegram)

  • Puerto 80/TCP y 443/TCP. Aunque es un protocolo propio, MTProto (usado por Telegram) a menudo se disfraza en puertos estandar como el 80 (HTTP) y el 443 (HTTPS) para evitar ser bloqueado por firewalls que solo permiten trafico web.
